Delivering Custom Fonts: FreakyStyle

This page is using a font called "FreakyStyle". It is sort of a far-out font, the likes of which are not seen on standard systems. Notice how we have embedded this custom font using the @font-face rule in CSS, that allows for implicit downloading of the font by the browser for use on the page. How did we do that?.

Font Selection: FreakyStyle

Freaky Style is a pretty unusual font, with a bold feeling to it; it has irregularly formed letters with dramatic angles. It's playful but still highly readable, ensuring it stands out as a custom font and isn't confused with any standard web fonts.

Why Choose a Custom Font?

Stand out with unique aesthetic: Using a unique font will give an unmistakable aesthetic to the web. In this case, FreakyStyle is fun and rather exaggerated, which can easily bring in good design.

Control Over Design: By using the embedded font, you ensure the page will appear as a whole in every browser and device, not dependent on what fonts the users have installed on their system.

Readability and Flexibility: FreakyStyle is kooky but readable; it will perform well for this assignment, which specified both weird-looking and readable.

Step-by-Step Process for Delivering the "FreakyStyle" Font

1. Creating Fonts

Freaky Style along with other fonts was typically created in design programs like Adobe Illustrator, FontForge, or Glyphs. These programs will enable the editing of letter-by-letter and can export a font to the .WOFF and .WOFF2 formats, among other web-compatible formats.

2. Host the Font File

Once the font is ready, you'll want to upload the font files, say freakystyle.woff2 to a web server or Content Delivery Network (CDN). For this example, we'll be hosting our font at https://yourdomain.com/fonts/freakystyle.woff2.

3. Use the @font-face Rule

The next CSS rule, @font-face, tells the browser to download the font from the server. Different formats are supplied for this, such as .woff2 for modern browsers and .woff for older ones, to guarantee that it would appear correctly on every platform.

4. Apply the Font to the Web Page

Well, once the custom font is defined using the code @font-face, we apply it on the whole page using CSS through the property font-family. In this case, it applies to the body, meaning all text appearing on the page would use FreakyStyle.

Detailed Process Description

Font Creation

FreakyStyle was designed to have full customisation on every letterform using a vector based application. Once created it was then exported to modern web friendly filetypes such as .woff2 and .woff for tiny file sizes and browser support.

Hosting the Font

That font was loaded onto some server or Content Delivery Network, or CDN-where the browser then pulled it from when the loading of the webpage occurred. In other words, the hosting server plays an important role in providing the font file at the time of page rendering .

The @font-face Rule

The rule @font-face plays the most important role in embedding fonts. In this case, we will provide two formats, .woff2 and .woff, to cover all browsers so that no problem happens when the browser shows up the font. Because of its smaller size, modern browsers use .woff2, and older browsers fall back to using .woff. When the web page comes up, it reads through the CSS and identifies the definition of the custom font. Then, it requests the font from the server and downloads it for application onto text.

Applying Fonts

Well, once the font is downloaded, it is used by the browser to render, based on the CSS rule set within the font-family property. For our example, we'll use the custom "FreakyStyle" font for all of the text on the page.

Fallback Fonts

We further define fallbacks here, such as sans-serif, in case the custom font doesn't load, so that text remains readable.
